單元四 效果的種類  
 
人氣：
 
 

觸發事件最好玩的地方，莫過於「效果」了。我們可以在「狀況」發生時，啟動其下的效果，就像單元二中所提到。

而效果的種類也滿多的，用法也很簡單，以下小弟就舉出幾個大家可能比較不熟悉的項目來說明：

貢品：

一般而言，貢品這個項目並沒有多大的作用，但是如果您與電腦對戰時，覺得電腦太肉的話，可以利用計時器 (狀況) ，每隔一段時間來給電腦一點資源 (來源者設為大地之母) 。不過請不要用這項功能來作弊喔！

啟動 / 關閉觸發事件：

在我們設定觸發事件時，有時會先將其關閉，等待其他事件發生後才將其啟動。例如一定要先問智者有關魔王的弱點，才能進行下一步，那麼我們就可以利用啟動觸發事件這個效果來啟動尚未啟動的觸發事件。而關閉觸發事件則是用來關閉一開始就已經啟動，而且是循環狀態下的觸發事件上。

建立物件：

建立物件也是一個相當容易了解的效果。它的特點是會憑空生出一個單位 (或建築) 而不會浪費您的資源。

指派物件：

指派物件就是指讓一個單位自動走到您指定的區域去。但是如果我們要指定的物件還沒創造出來呢？也就是我們用上述的建立物件所創造的單位要使用這個效果時怎麼辦呢？這時，我們就必須了解什麼是「區域中的物件」。

摧毀物件 / 移除物件：

摧毀物件和移除物件最大的不同是，摧毀物件是讓物件死亡；而移除物件則是將物件將從地圖上移除、消失。

顯示指令：

顯示指令就是用來在螢幕上顯示訊息用的。這裡要注意的是，在顯示指令中，有個「數量」的設定欄，這是用來設定訊息要顯示的行數。一般遊戲畫面能容納五到八行，所以如果您是製作對話系統，請記得將數量改成不一樣的數字，這樣一來對話才不會被蓋掉。詳細的例子請到「檔案下載」區下載小弟的拙作。

另外，網友 Ladaga 提供了這樣的寶貴資料：

您玩過AOKEX的戰役嗎？是否發現他的對話跟以前AOK不一樣，原本是只有白色的，現在變成彩色的了呢！

要讓你做的地圖產生文字訊息，可使用觸發事件的「顯示指令」及「送出交談訊息」。但是，這兩個裡面並沒有看到任何改變文字顏色的選項，但是為什麼AOKEX的戰役裡會出現彩色的字呢？

我在修改language_x1.dll時找到答案了！因為原版中文版的language_x1.dll中文檔做的不太好，中國人的火箭技術弩砲攻擊力+4，他把「弩砲」二字給人家翻成「蠍子」，所以我用「pebble」這個程式去抓那個檔的字串表，去改他。改也改好了，而且發現了一件事....... 在此檔裡面有存著各關卡的中文對話，發現他每一句前面都加著一個，<xxxx> ，這個東東。而且， xxxx ，都是某種顏色的英文。我就跑回遊戲去做實驗，結果實驗成功。

結論：只要在使用「顯示指令」、「送出交談訊息」的時候，在文字的開頭加上：<blue>、<red>、<yellow>、<purple>、<aqua>、<green>、<orange>、<white>就可以讓那段文字變成所指定的顏色，裡面的英文字要用大寫。 不知道這個消息對你們有沒有幫助，...還是你們早就知道了，那就當成野人獻曝吧！

凍結單位：

這個效果是用來讓指定的單位定住不動，就算對其攻擊也不會有任何反應。要使用這個效果以前，請先將電腦的性格設定為「無」。

另外在 AOKEX 中，還新增了幾個新效果，可以用來編輯物件。因為小弟用的是 MYTH 版，所以是英文的，大家就先將就一下吧。

以下就請看小弟說明：

Damage Object (損害物件) ：

這會讓您的物件 (單位或建築) 扣掉「數量」欄中所指定數字的生命值。對於英雄雖然也有用，但英雄會自動回復生命值，所以作用不大。

Place Foundation (放置地基) ：

與建立物件 (建築) 相同，但僅是放置一個地基，但不會浪費您現有的資源。

Change Object Name (更改物件名稱) ：

這是網路上常被問到的功能，這可以用來改變物件 (單位、建築、英雄) 的名稱。我們只要在訊息欄上輸入您想改的名稱就可以了。

Change Object HP (更改物件生命值) ：

這個效果會將原物件的生命，加上您指定的數字。如劍兵勇士一開始的生命值若為 70 ，而您在數量欄中輸入 1000 的話，那麼其生命值就會是 1070 。這個效果也可以用在建築物上。

Change Object Attack (更改物件攻擊力) ：

設定方式與 Change Object HP 相同，只不過其加成的是物件的攻擊力。對沒有攻擊力的物件 (如房子) 沒用。

以上三種最好的設定法是一開始就新增一個觸發事件「變更物件、否、開、否」，也不需設定狀況，直接新增上述三種效果，這樣您的單位才會一開始就改變。

Stop Unit (停止單位) ：

這個就是讓行動中的單位停止的效果。與凍結單位不同之處是在於該單位仍對周遭有反應。

觸發事件就教到這裡，下一個單元是一些應用上的範例。如果您對效果還不熟，就請您看完本單元後，上機實習一下。

區域中的物件： (非狀況或效果)

有些效果中，我們可以看到「設定單位」和「設定區域」兩種設定法。當然設定單位就必須是目前已經存在地圖上的物件；而設定區域則是可以一次選取一群單位。

不過我們也可以在單位尚未出現時設定好區域，到時候只要將創造出來的物件指派到該區域，再觸發該事件就行啦！

還有一點要注意，當您為一個效果設定區域時，也要注意作用在這個區域上的物件類型，以免所有的單位都會一起被您指派。像來源遊戲者、物件類型清單 (單位、建築、英雄、其他) 與物件類別 (文明、軍事) 等，這一定要設對。 (不過旁邊還有物件清單什麼的，它們是要做什麼用的，小弟就不清楚了。)

例如我們現在有三個村民和兩個民兵在一個作用區域上，而我們只想讓民兵被指派到某一區域，那麼來源遊戲者就是「遊戲者一」，物件類型清單就是「單位」，物件類型則是「軍事」。
